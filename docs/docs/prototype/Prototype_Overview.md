# Council of 1000 – Playable Prototype Overview
### Vertical Slice MVP Definition

---

# 🜂 Purpose

The purpose of this prototype is to create the **first playable interaction** of Council of 1000. This vertical slice is designed to prove the core gameplay loop is engaging before expanding into blockchain, multiplayer, or economic systems.

This prototype focuses only on:

• Player onboarding  
• Champion creation  
• Ability drafting  
• Combat flow  
• Visual battle presentation  
• Player engagement hook  

This is **NOT** a full game build.

---

# 🜁 Prototype Philosophy

This prototype follows three design principles:

### 1 — Interaction over theory
Playable experience takes priority over additional documentation.

### 2 — Minimum viable fun
The goal is proving the gameplay loop feels rewarding.

### 3 — Expandable architecture
All systems should be built in a way that allows future expansion.

---

# 🜄 Prototype Core Loop

The player experience follows this sequence:

Landing Page  
→ Faction Selection  
→ Champion Creation  
→ Ability Draft  
→ Champion Identity Generation  
→ Tutorial Battle  
→ Victory Outcome  
→ Player Hook

---

# 🜃 Prototype Goals

The prototype must successfully demonstrate:

• Champion creation feels meaningful  
• Draft choices feel impactful  
• Combat flow is understandable  
• Visual battle playback is engaging  
• Player feels ownership of their champion  
• Tutorial creates desire to continue

If these goals are achieved, the prototype is successful.

---

# Champion Creation Rules

Players must choose:

Random Champion  
or  
Custom Champion

This choice is permanent.

Custom champions receive a fixed pool of Champion Points to allocate.

All choices are irreversible.

This reinforces identity ownership and decision weight.

---

# Ability Draft Rules

Players will complete:

3 Draft Rounds

Each round presents:

Pick 1 of 3 abilities

Abilities are randomly selected from prototype pools.

Player selections are permanent.

No rerolls in prototype.

---

# Tutorial Battle Purpose

The tutorial battle serves to:

Teach turn flow  
Demonstrate abilities  
Show combat visuals  
Create emotional investment  

The tutorial is scripted to ensure player victory.

This is intentional onboarding design.

---

# Definition of Prototype Completion

The prototype is considered complete when a player can:

Select faction  
Build champion  
Complete ability draft  
Enter tutorial battle  
Experience combat playback  
Win encounter  
Reach completion screen  

No backend persistence required.
