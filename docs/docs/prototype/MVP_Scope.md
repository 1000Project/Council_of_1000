# Council of 1000 – MVP Scope Definition
### Prototype Development Boundaries

---

# 🜂 Purpose

This document defines exactly what the prototype includes and what it intentionally excludes. Its purpose is to prevent scope creep and maintain development focus.

Any feature not listed under **Included Systems** is automatically considered **out of scope** unless explicitly added later.

---

# 🜁 MVP Definition

The MVP (Minimum Viable Prototype) is defined as:

A playable vertical slice demonstrating champion creation, drafting decisions, and a scripted combat experience.

The MVP is not intended to represent the full game.

It exists only to validate:

• Player engagement  
• System interaction  
• Core gameplay loop  
• Visual presentation direction  

---

# 🜄 MVP Development Priorities

Development priority order:

Priority 1:
Champion Creation Experience

Priority 2:
Ability Draft Interaction

Priority 3:
Combat Simulation Flow

Priority 4:
Battle Presentation

Priority 5:
Completion Hook

Anything outside these priorities must not delay prototype completion.

---

# 🜃 Included MVP Features

The MVP will contain:

## Player Entry

Landing page  
Start prototype button  

## Faction Selection

6 faction choices:

Order  
Entropy  
Knowledge  
Power  
Void  
Balance  

Faction selection is permanent.

Prototype impact:
Identity only.

## Champion Creation

Two paths:

Random Champion  
Custom Champion

Custom Champion receives Champion Points.

Stats available:

Strength  
Speed  
Willpower  
Intellect  
Resilience  
Influence  
Luck  

Stat allocation is permanent.

## Ability Draft System

3 Draft Rounds.

Each round:

Player selects 1 ability from 3 randomized options.

Final result:

3 abilities total.

No rerolls.
No replacements.

## Champion Identity Generation

After drafting:

Champion summary generated including:

Faction  
Stats  
Abilities  
Champion designation  

Identity card displayed to player.

## Tutorial Battle

Player enters scripted encounter.

Battle includes:

4–6 turns

Player uses abilities.

Enemy performs scripted actions.

Battle always results in player victory.

Purpose:

Teach flow
Create engagement
Demonstrate visuals

## Combat Mechanics (Prototype Level)

Basic systems only:

Health values  
Damage application  
Ability activation  
Turn order display  

No advanced systems.

## Arena Viewer

Battle presentation includes:

Health bars  
Damage numbers  
Turn indicators  
Basic animation effects  
Ability activation visuals  

No advanced graphics required.

## Completion Screen

Player receives:

Victory message  
Champion summary  
Possible teaser message

Example:

"Your champion is ready for the Council."

Purpose:

Retention hook.

# Explicitly Excluded MVP Features

These systems are not allowed in prototype:

Blockchain integration  
NFT minting  
Wallet connection  
Token rewards  
PvP  
Online multiplayer  
AI opponents  
Leaderboards  
Matchmaking  
Persistent storage  
Account systems  
Card packs  
Inventory systems  
Crafting  
Progression trees  
Guild systems  
Economy systems  
Ranking systems  
Season systems  
Map exploration  
Narrative campaign  

If it delays prototype, it does not belong in MVP.

---

# Technical Constraints

Prototype should:

Run in browser  
Require no login  
Require no backend  
Use local state only  
Focus on responsiveness  

Acceptable:

Mock data  
Scripted combat  
Simplified logic  

Unacceptable:

Infrastructure dependencies
Complex networking
Production scaling concerns

---

# MVP Completion Checklist

Prototype is complete when:

Player can start prototype

Player selects faction

Player creates champion

Player drafts abilities

Player views champion summary

Player enters tutorial battle

Battle plays correctly

Player wins

Player reaches completion screen

If these conditions are met, MVP is finished.

---

# MVP Kill Criteria

The prototype should be considered successful even if:

Graphics are basic  
Animations are simple  
Abilities are few  
UI is minimal  

Failure is only defined as:

Prototype not playable.

Playable always beats perfect.

---

# MVP Expansion Trigger

Expansion begins only after:

Playable build exists.

Then possible additions:

More abilities  
More factions  
Better visuals  
Balance passes  
Advanced combat rules  

Never before.

---

# Development Discipline Rule

If a feature does not directly improve:

Champion creation
Draft interaction
Combat experience

It must be postponed.
