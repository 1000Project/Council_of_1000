# ğŸœ‚ Player Priority

Throughout the turn, **priority** alternates between the two players.  
A player may only perform actions when they have priority.

Priority is passed:

- after an action  
- after a trigger resolves  
- at the end of each sub-phase  
- before combat declaration  
- after combat resolution  
- before end-of-turn cleanup  

This system keeps interactions fair and strategic without overwhelming complexity.

# ğŸœ Realm Influence on Turn Structure

Each Realm interacts with the turn differently:

### **Dominion**  
Thrives in Main Phase aggression; pushes tempo early.

### **Solace**  
Leverages End Phase to restore, cleanse, or shield.

### **Veil**  
Controls priority windows; thrives on reactions.

### **Genesis**  
Establishes value engines and creation loops during Main Phase.

### **Ruin**  
Triggers decay effects during Draw or End Phases.

### **Ascendance**  
Synergizes abilities across multiple phases for mastery combos.

### **Paradox**  
Alters timing rules and probability during Combat Phase.

These differences make Realms feel distinct without altering the universal turn structure.
